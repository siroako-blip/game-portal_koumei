## 2026-06-16 — ポーカー（テキサスホールデム）を新規実装

- 9つ目のゲームとして **Poker（テキサスホールデム / フルベット / 2〜6人）** を追加。要件定義のうえ既存パターン（多人数型 coyote）に準拠して実装。
- `app/poker/logic.ts` に純粋ロジックを実装: 52枚デッキ、ホール2枚＋コミュニティ5枚、プリフロップ/フロップ/ターン/リバーの4ベッティングラウンド、フォールド/チェック/コール/レイズ/オールイン、**役判定（7枚から最良5枚・A-5ホイール対応）**、**サイドポット分配**、ディーラーボタン移動、合意制リスタート。
- DB(`poker_games`)・`useRealtime`・ロビー・対戦画面（カジノ風の緑×金テーマ）・ポータル/ルールブックを追加。伏せ札は既存方針どおりクライアント表示で隠す。
- ロジックは一時transpileしてユニットテスト済み（役判定・2〜6人のチップ保存・オールイン時のサイドポット）。**マイグレーション `20260616000000_create_poker_games.sql` は本番 Supabase で実行＋Realtime有効化が必要。**

## 2026-06-15 — ルールブックを Lost Cities 流に全ゲーム統一・内容ズレを修正

- 共有 `components/RuleBook.tsx` を **Lost Cities のルールモーダル風**（amber基調・絵文字付き見出し・「理解した！」フッター・右上「📜ルール」ボタン）に作り替え、**全8ゲームを共有コンポーネントに統一**。

  - coyote: チャレンジ宣言が実装は「Coyote!」なのにルール文が「Midnight!」だった誤記を修正（旧テーブル名 midnight の名残）。マイナスカード枚数（−10×2・−20×2）と MAX=0 の説明も実装に合わせて精密化。
  - loveletter: 王子(5)で姫を捨てさせた相手は「引き直しもできず脱落」、山札枯渇時の同点は「捨て札合計」でタイブレーク、を追記。
- lostcities / hitblow / wordwolf / ito / deepsea / nothanks は内容自体は実装と一致していたため、見出し・体裁のみ統一。
